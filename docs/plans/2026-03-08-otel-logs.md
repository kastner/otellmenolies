# OTLP Log Ingest Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Accept OTLP log exports and append decoded log entries to daily JSONL files under the local data directory.

**Architecture:** Extend the vendored OTLP proto loading and receiver shape to include the LogsService, decode resource log payloads into plain log records, and pass them through the existing buffered ingest path into a small append-only file store. Keep the storage local-first and file-based, with `data/logs` as the default root and `LOGS_DIR` as an optional override.

**Tech Stack:** Bun, TypeScript, Vitest, vendored OpenTelemetry protobufs, Node fs/path
